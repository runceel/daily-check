- `1.6.0` リリース後の **リグレッション** であり、experimental トレーシングと組み合わせる利用者はパッチ待ちまで `1.5.0` 固定が回避策。

OAuth 認証を必要とする MCP ツールを Foundry agent で利用すると、Foundry playground では Consent Link メッセージが流れて認証可能だが、Agent Framework + AGUI を経由するとレスポンスストリーム上に何も受信できないため、エンドユーザーが認証フローへ入れない。原因は当該イベントが `OpenAI.Responses.UnknownResponseStreamEvent` (内部型) として扱われ、`RawRepresentation.Kind` (`oauth_consent_response`) を外から判別する手段が無いこと。

- 起票者はミドルウェアで `RawRepresentation` を覗くワークアラウンドを実装しているが、内部 API 依存のため公式 API 化を要望。
- 提案: `ToolCallContent` または middleware から `oauth_consent_request` を識別できる強い型 (e.g. `OAuthConsentRequestContent`) を導入する案。
- 現時点 `triage` 段階で、メンテナの正式コメント無し。

**関連 PR**

- なし (機能要望、今後のロードマップ候補)
