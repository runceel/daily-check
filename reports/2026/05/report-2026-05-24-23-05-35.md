##### [#6052](https://github.com/microsoft/agent-framework/pull/6052) — Python: feat(bedrock): implement native structured output support via Converse API

- 作者: karthik-0306 / 状態: OPEN / 作成日時 (UTC): `2026-05-23 09:25:41`
- ラベル: python

**変更概要**

BedrockChatClient が MAF クライアント群で唯一構造化出力をサポートしていなかった (内部で `response_format=None` をハードコードしてユーザ指定スキーマを黙って捨てていた) 問題を解消する PR。AWS Bedrock Converse API が 2026 年 2 月 4 日に **`outputConfig.textFormat` 経由のネイティブ構造化出力**を GA したのに合わせ、その配線を行う。これにより Anthropic / OpenAI / Gemini と同じく Pydantic モデルまたは JSON Schema を `response_format` 引数で渡せるようになる。

**変更ファイル (主要)**

```
python/packages/bedrock/agent_framework_bedrock/_chat_client.py  (+144 / -12)
python/packages/bedrock/tests/test_bedrock_structured_output.py  (+268 / -0, new)
```

**コミットレベルの詳細**

- `BedrockChatOptions` の `response_format=None` 上書きを撤去し、親 `ChatOptions` の値が自然に流れるよう修正。
- 新ヘルパー `_prepare_output_config()` を追加。Pydantic モデルクラスまたは OpenAI 形式の dict スキーマを受け取り、Converse API 仕様どおりの `outputConfig.textFormat.structure.jsonSchema` 形に変換する。
- `_set_additional_properties_false()` を再帰的に適用し、全 `object` 型に `additionalProperties: false` を付与 (AWS の strict schema 要件)。元 dict を破壊しないよう `copy.deepcopy` で防御。
- `_process_converse_response()` / `_build_response_stream()` を経由して `ChatResponse.value` に検証済み Pydantic インスタンスが lazy hydrate される。
- `botocore.ClientError` を捕捉し、`outputConfig` を含む `ValidationException` が返った場合 (Claude 3.x / Nova / Llama 等の非対応モデル) は明示的な `ValueError` に変換。フォールバックで非構造化テキストへ静かに戻さない。

```python
# Bedrock 経由でも他クライアントと同じ書き方で構造化出力が取得可能になる:
class Order(BaseModel):
    sku: str
    qty: int

resp = await bedrock_chat_client.get_response(
    "Order: 3 widgets, SKU W-1",
    response_format=Order,
)
order = resp.value  # Order インスタンス
```

**関連 Issue / Discussion**

- Fixes [#5966](https://github.com/microsoft/agent-framework/issues/5966)

**既存利用者への影響**

破壊的変更なし。`response_format` を指定しなかった場合のリクエストは従来と完全同一であり、既存コードは無変更で動作する。Claude 3.x / Nova / Llama 等の `outputConfig` 未対応モデルを使っている場合は、構造化出力を有効化した瞬間に `ValueError` が投げられる仕様変更となる点だけ注意 (黙ってテキストへフォールバックしない)。

#### 主要な変更点

- **⚠ 破壊的変更**: [#17424](https://github.com/microsoft/aspire/pull/17424) で Kubernetes Ingress 用 `WithRoute(...)` を **`WithPath(...)`** に改名 (path-only / host+path の両オーバーロード)。内部型 `IngressRouteConfig` / `Routes` も `IngressPathConfig` / `Paths` に改名。Gateway API 側は意味的に正しいため `WithRoute` を維持し、共有していた `IngressPathType` を分離。ATS (TypeScript) からは `withIngressPath` / `withIngressHostAndPath` がエクスポートされる。
- **⚠ 破壊的変更**: [#17382](https://github.com/microsoft/aspire/pull/17382) で JavaScript パッケージスクリプトの公開 API を **`PublishAsPackageScript`** に統一 (C# は `PublishAsPackageScript`、TypeScript は `publishAsPackageScript`)。引数も `startScriptName` → `scriptName` にリネームし、Verify スナップショット名や agent guidance も追随。npm / pnpm / Yarn / Bun を統一的に扱うための前提整理。
- [#17416](https://github.com/microsoft/aspire/pull/17416): **`AddBunApp` を新規追加**。`bun server.ts` のような Bun ネイティブ起動 (package script 経由でない) を `AddNodeApp` と同形で表現できる。`Aspire.Hosting.JavaScript` パッケージに追加。
- [#17400](https://github.com/microsoft/aspire/pull/17400): TypeScript AppHost の **生成 async メソッドが thenable wrapper を返す**ようコード生成を更新。`builder.addContainer(...).withReference(...)` 系のチェーンが async API を跨いでもダブル await 無しで書けるようになる。
- [#17335](https://github.com/microsoft/aspire/pull/17335): **AppHost 起動時間を最適化**。CLI が AppHost を既にビルド済みのときに `dotnet run --no-build --project ...` の経路をスキップし、SDK が解決した `RunCommand` / `RunArguments` / `RunWorkingDirectory` を使って apphost 実行ファイルを直接起動する。launch profile / `dotnet exec` 系の挙動はランタイム側でエミュレート。
- [#17443](https://github.com/microsoft/aspire/pull/17443) / [#17422](https://github.com/microsoft/aspire/pull/17422): CLI の Spectre / banner 描画が stdout リダイレクト時や非対話 Windows セッションで crash していた問題を修正。CI でのログ収集や`aspire --version` のスクリプト利用が安定化。
- [#17451](https://github.com/microsoft/aspire/pull/17451): `CliSmokeTests.MainReturnsExpectedExitCode` の intermittent failure (`Azure.Monitor.OpenTelemetry.Exporter` の `RateLimitedSampler` が `microsoft.sample_rate` を重複登録) を解消するため、当該テストを直列化。
- [#17388](https://github.com/microsoft/aspire/pull/17388): Dashboard の Traces ページに **duration filter** を追加。
