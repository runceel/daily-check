## 0. エグゼクティブサマリー

Microsoft Build 2026 のメッセージは、ひとことで言えば **「アシスタントから自律エージェント (Autonomous Agents) へのプラットフォームシフト」** です。Satya Nadella 氏と Mustafa Suleyman 氏 (Microsoft AI CEO) が登壇したキーノートでは、以下の柱が示されました。

1. **Microsoft Foundry** が「本番運用エージェント」のフルスタックプラットフォームへ (Microsoft Agent Framework 1.0 GA / Hosted Agents GA 間近)
2. **Microsoft IQ** (Work IQ / Foundry IQ / Fabric IQ / Web IQ) を「エージェント時代のコンテキスト基盤」として一般提供
3. 自社モデル群 **MAI ファミリー**を 7 機種同時発表 (Thinking-1, Code-1-Flash, Image-2.5, Voice-2, Transcribe-1.5 ほか)
4. **GitHub Copilot** の "プラットフォーム化": **Copilot SDK GA**, **Copilot デスクトップアプリ (Technical Preview)**, **Copilot CLI に Build 2026 アップデート**, **Visual Studio が Copilot SDK ベースへ**
5. **Windows をエージェントの実行基盤に**: MXC (Microsoft Execution Containers), OpenClaw on Windows, Windows 365 for Agents, WSL containers, Intelligent Terminal
6. インフラ層では **Maia 200** AI アクセラレータ本番投入と、**Project Solara** (chip-to-cloud のエージェント新プラットフォーム)、量子チップ **Majorana 2** 発表
7. 公式発表手段が刷新: 従来の "Book of News" PDF は廃止、**Microsoft Build Live** というライブブログ形式に変更

## 3. AI アプリ開発関連の発表 (重点解説)

### 3.1 Microsoft Foundry (旧 Azure AI Foundry) — エージェントの本番運用プラットフォーム

公式ライブブログ "Microsoft Foundry updates: agents you deploy, improve and trust in production" を中心に整理します ([Live Blog 該当エントリ](https://news.microsoft.com/build-2026-live-blog))。Foundry は **Build / Ground / Operate / Reach** の 4 層で再整理されました。

#### Build 層
- **Microsoft Agent Framework (MAF) 1.0 GA**: Python / .NET 両対応。production-ready なエージェントオーケストレーション。Skills, Context, Memory, Middleware が first-class。GitHub Copilot SDK や Claude Agent SDK のエージェントを named participant として MAF ワークフローに組み込み可能。詳細: [Microsoft Agent Framework at BUILD 2026](https://devblogs.microsoft.com/agent-framework/microsoft-agent-framework-at-build-2026/), [GitHub: microsoft/agent-framework](https://github.com/microsoft/agent-framework)。
- **Hosted Agents (Foundry Agent Service) — 数週間内に GA**: per-session sandboxing、sub-100 ms cold start、idle 課金ゼロのフレームワーク非依存ランタイム。
- **Foundry Toolboxes (Preview)**: Web Search / File Search / MCP / OpenAPI / A2A プロトコルを統一アクセス。
- **Fireworks AI on Foundry (GA)** と **Managed Compute (Private Preview)**: 専用 GPU で OSS モデルを単一 Azure エンドポイント経由で利用可能。

#### Ground 層
- **Foundry IQ knowledge bases (GA)**: Work IQ / Fabric IQ / File Search / Azure SQL / MCP を 1 つの SLA バックの retrieval エンドポイントに統合。
- **Microsoft World Grounding**: 社内データに加えてライブな外部情報へ拡張。
- **Procedural Memory (Preview)**: 何をするかだけでなく「やり方 (how)」を複数実行をまたいで学習。

#### Operate 層
- **Tracing & Evaluation for hosted agents (GA)**: 単一 OpenTelemetry パイプライン、トレースに評価を紐付け。
- **Agent Optimizer (Preview 近日)**: シグナルから prompt / tools / skills / context の候補改善案をランク付け生成、diff・監査・1-click rollback 対応。
- **Adaptive Evaluations (Preview)**: ポリシーを自動テストに変換。
- **Agent Control Specification (Preview)**: production でのエージェント挙動の宣言と強制を、Foundry / Microsoft Agent Framework / LangChain で共通化。

#### Reach 層
- **Microsoft 365 / Teams への 1-click publishing (来月 GA)** — テナントのアイデンティティ・ポリシーが自動的に流れる。

参考: [Microsoft Foundry Blog (公式)](https://devblogs.microsoft.com/foundry/), Build 編まとめ "What's new in Microsoft Foundry | Build Edition" (<https://devblogs.microsoft.com/foundry/whats-new-in-microsoft-foundry-build-2026/>)。

### 3.3 Microsoft IQ — エージェント時代の共有インテリジェンス基盤

(出典: [Live Blog "Microsoft IQ" エントリ](https://news.microsoft.com/build-2026-live-blog))

- **Microsoft IQ** (GA): Work IQ / Fabric IQ / Foundry IQ を統合し、GitHub Copilot, Microsoft Foundry, Copilot Studio から横断アクセス。「1 度作って、エージェントが動くあらゆる場所で使い回す」モデル。
- **Work IQ (GA, 今月)**: 会話・ドキュメント・ワークフローにまたがる semantic 理解レイヤ。
- **Foundry IQ (GA)**: 企業ナレッジと Web を Web IQ 経由で推論。
- **Fabric IQ ontology (Preview)**: People / Data / Workflows / Operations の関係を表現する共有 business semantics。
- **Web IQ (限定アクセス開始)**: Microsoft Copilot と ChatGPT のグラウンディングを既に支える、AI-native なグラウンディング API 群 (Web / News / Image / Video の発見・ランキング・抽出・パッケージ)。レイテンシ・回答品質・トークン効率で新基準を打ち出すと主張。<https://aka.ms/WebIQ>。

### 3.4 MAI モデルファミリー (7 機種同時発表)

公式トランスクリプト: <https://microsoft.ai/news/microsoft-build-2026-mai-keynote-transcript/>

| モデル | 種別 / 用途 | 主な特徴 |
|---|---|---|
| **MAI-Thinking-1** | テキスト reasoning | 35B active param MoE / 256K コンテキスト / **AIME 25 で 97%**, **SWE Bench Pro で 53%** (Opus 4.6 並び) / クリーンなライセンスデータ・蒸留なし |
| **MAI-Code-1-Flash** | コーディング (VS Code / Copilot CLI 向け) | わずか 5B パラメータで **SWE Bench Pro 51%** / Haiku より低コスト / VS Code の既定モデルの 1 つとして当日ロールアウト |
| **MAI-Image-2.5 / Flash** | 画像 (T2I + I2I) | 編集系リーダーボード 2 位 (Nano Banana 2 を上回る), PowerPoint・OneDrive に統合、Foundry でも提供 |
| **MAI-Transcribe-1.5** | 文字起こし | 43 言語で SOTA、Gemini や OpenAI モデルを上回る精度、最大 5 倍速、Copilot / Teams / GitHub / Dynamics 365 Contact Centre に統合 |
| **MAI-Voice-2 / Voice-2-Flash** | 音声生成 | 15 言語、fine-grained な感情制御、低遅延 Voice Agent 用に Flash 版 |

- 提供チャネル: Microsoft Foundry / MAI Playground / Open Router / Fireworks / Baseten (重み調整可)
- 安全機構: 音声複製防止、出力ウォーターマーク、過剰拒否削減
- **Mayo Clinic と「医療向けフロンティアモデル」を共同開発**することを発表

### 3.5 Frontier Tuning と Microsoft Scout / Discovery / Rayfin

(出典: [Live Blog 該当エントリ群](https://news.microsoft.com/build-2026-live-blog))

- **Frontier Tuning (Early Partners)**: 自社テナント内で動作するドメイン特化チューニング。エンタープライズ RL 環境 (RLE) を同梱。「あなたのワークフロー・データ・スタイルを学習する」ためのもの。重要メッセージとして「他社のように共有モデルからインテリジェンスを借りるのではなく、**学習成果はあなただけの "moat" になる**」。McKinsey との早期実験では「品質で GPT-5.5 を上回り、コストは 1/10」との主張。
- **Microsoft Scout** (Frontier Enterprise 早期実験提供): Microsoft 初の "Autopilot" エージェント。常時稼働で Teams / Outlook / OneDrive / SharePoint および端末ローカルで自律実行。OpenClaw 上で動作し、各エージェントは独立した **Entra ID** を保持。Work IQ をコンテキストエンジンとして利用。
- **Microsoft Discovery (GA)** + 新しい Discovery App (Early Preview): 研究開発向け agentic AI プラットフォーム。仮説生成と検証ループに特化した Discovery Engine。GitHub Copilot アカウントがあればローカルでも実行可。<https://azure.microsoft.com/en-us/solutions/discovery>。
- **Rayfin (Preview)**: コーディングエージェントが秒で生成したアプリの「本番化」ボトルネック (DB / 認証 / ストレージ / アクセスポリシー) を、OSS の SDK + CLI で 1 コマンド化。**Microsoft Fabric** にマネージドサービスとして展開、データは OneLake にそのまま着地 (ETL 不要)。Replit との提携で「Replit でビルド → Rayfin でデプロイ」のフロー実現。
- **MDASH** (前週発表) のプッシュ — マルチモデルな agentic セキュリティシステム ([Security Blog](https://www.microsoft.com/en-us/security/blog/2026/05/12/defense-at-ai-speed-microsofts-new-multi-model-agentic-security-system-tops-leading-industry-benchmark/))。

## 4. GitHub Copilot 関連の発表 (重点解説)

GitHub / Copilot のアップデートは Build 2026 の最大の目玉領域です。ライブブログでも "The GitHub Copilot app for agentic development" として大きく扱われました。

### 4.1 GitHub Copilot SDK が GA (6 月 2 日付)

公式チェンジログ: <https://github.blog/changelog/2026-06-02-copilot-sdk-is-now-generally-available/>

- **GitHub Copilot SDK** が一般提供開始。GitHub Copilot の agent runtime (planning / tool invocation / file edits / streaming / multi-turn session) を **API 経由でプログラマブルに呼び出せる**。
- リポジトリ: <https://github.com/github/copilot-sdk>
- **対応言語 6 種**: Node.js/TypeScript (`npm i @github/copilot-sdk`), Python (`pip install github-copilot-sdk`), Go, **.NET (`dotnet add package GitHub.Copilot.SDK`)**, **Rust (GA 新規)**, **Java (Maven/Gradle, GA 新規)**
- 主要機能:
  - **カスタムツールと MCP** (built-in `grep` / `edit_file` の上書きも可)
  - **System prompt セクション単位のカスタマイズ** (identity / tone / tool instructions / safety rules)
  - **OpenTelemetry tracing** (W3C trace context, CLI 起動 / JSON-RPC / セッション / ツール実行を横断)
  - **柔軟な認証**: GitHub OAuth / GitHub Apps / 環境トークン / **BYOK** (OpenAI, Microsoft Foundry, Anthropic ほか)
  - **クラウド / リモートセッション**: repository メタデータ付きクラウドセッション、URL によるリモートセッション公開
  - **Hook system**: pre/post tool use, session start, MCP tool calls, permission requests を傍受
- public preview からの主な変更点: Rust SDK 新設、複数クライアントが同一セッションにツール / 権限を寄与可能、Slash コマンドと対話入力プロンプトを全 SDK で標準化、API 安定化、診断改善
- **価格**: 既存 Copilot サブスクライバ (Free 含む) は無料、非 Copilot ユーザは **BYOK** 経由で利用可
- ドキュメント: <https://docs.github.com/copilot/how-tos/copilot-sdk>、レシピ集: <https://github.com/github/awesome-copilot/tree/main/cookbook/copilot-sdk>

> **重要度**: Visual Studio 自身も「今後の AI 統合基盤を Copilot SDK に再構築する」と発表しており、SDK は単なる外部公開 API ではなく、**Microsoft / GitHub 共通の agent runtime 基盤**として位置付けられています ([Visual Studio Blog](https://devblogs.microsoft.com/visualstudio/whats-coming-next-in-visual-studio-our-microsoft-build-2026-announcements/))。

### 4.2 GitHub Copilot App (デスクトップアプリ) が Technical Preview

公式チェンジログ: <https://github.blog/changelog/2026-05-14-github-copilot-app-is-now-available-in-technical-preview/>

- GitHub Issue / PR / プロンプト / 過去セッションから直接 **agentic 開発セッションを開始**できる、GitHub-native なデスクトップアプリ。
- ライブブログでは「Under the hood, the GitHub Copilot app uses **git worktrees**, so each session has its own space with its own branches, files, conversations and task state」と説明され、本セッションが動作している Copilot Workspace と同種の機能が示唆されています ([Live Blog "The GitHub Copilot app for agentic development"](https://news.microsoft.com/build-2026-live-blog))。
- 主な機能:
  - **GitHub コンテキストから開始**: Issue / PR / 受信箱から直接セッション化、issue 詳細 / repo state / レビューコメント / checks が連動
  - **集中セッション**: 各セッションが自前の branch / files / conversation / task state を持つ。pause / resume 対応、複数セッション並列、ワークフロー自動化 (triage / dependency updates / release notes など)
  - **Steer & Validate & Ship**: plan と diff レビュー、組み込みターミナル・ブラウザでの検証、PR への移行、**Agent Merge** (review コメント対応 → 失敗チェック修正 → 条件達成で merge まで自律)
- 提供範囲: Copilot Pro / Pro+ は Early Access サインアップ可、Business / Enterprise は週内ロールアウト (組織で previews と Copilot CLI policy 有効化が前提)。
- 早期アクセス: <https://gh.io/github-copilot-app>、リポジトリ: <https://gh.io/github-copilot-app-repo>、ドキュメント: <https://gh.io/github-copilot-app-docs-main>

### 4.4 Visual Studio 2026 と Copilot 統合

公式ブログ (Mads Kristensen): <https://devblogs.microsoft.com/visualstudio/whats-coming-next-in-visual-studio-our-microsoft-build-2026-announcements/>

「Code is an asset, not just an artifact」というメッセージのもと、以下を発表:

- **Agents that participate in the work** (BRK207 で実演): デバッガ / プロファイラ / テストツールと連携し、root-cause / パフォーマンス障害特定 / リグレッション防止のテスト生成を実施するエージェント。
- **Pre-build error / warning チェック**: Error List のエラーをビルド前に検出し、無駄なビルドを排除。
- **AI-assisted merge conflict resolution**: 自動マージではなく、コンフリクトの理解と意思決定支援を提供。
- **Copilot modernization の拡張** (この夏):
  - **Web Forms → Blazor** マイグレーション
  - 既存アプリへの **.NET Aspire** 追加 (cloud-ready なオブザーバビリティ / オーケストレーション)

## 5. Windows プラットフォーム — エージェントの実行基盤

公式 Windows Developer Blog: <https://blogs.windows.com/windowsdeveloper/2026/06/02/build-2026-furthering-windows-as-the-trusted-platform-for-development/>

### 5.1 開発者向け Windows 11 改善

- **Coreutils for Windows (GA)**: GNU Coreutils 互換を Rust で再実装 (uutils 由来) し Windows 上で native 実行。<https://github.com/microsoft/coreutils>
- **WSL containers (Public Preview 間近)**: Linux コンテナを Windows 上で第一級に扱う。`wslc.exe` CLI と API を提供。エンタープライズではポリシーベースの管理。OSS リポジトリ: <https://github.com/microsoft/wsl>
- **Windows Developer Configurations (GA)**: WinGet 設定ファイル `dev-config.winget` で WSL / PowerShell 7 / Git / GitHub CLI / VS Code / Python を 1 コマンドで導入。<https://github.com/microsoft/WindowsDeveloperConfig/>
- **Windows 365 with Developer Configuration (Public Preview)**: 同じ設定を Cloud PC で。
- **Windows Development Skills (GA)**: WinUI3 skills + winapp CLI でエージェントが Windows ネイティブアプリ開発を実行できる構造化ナレッジ。<https://aka.ms/winui-skills>
- **Intelligent Terminal (Experimental)**: Agent Communication Protocol (ACP) でエージェントを Windows Terminal に統合。失敗コマンドのコンテキストを自動収集し修正提案。GitHub Copilot もデフォルトで利用可。

### 5.2 エージェント実行基盤

- **Microsoft Execution Containers (MXC) SDK (Early Preview)**: 「このエージェントは何にアクセスできるか」をポリシーで宣言し、Windows OS 自身が runtime で強制。Files / Network 等に対し、リスクに応じて isolation の濃淡を動的合成。
- **OpenClaw on Windows (Alpha, OSS)**: MXC 上で動く。OpenClaw のノード / ゲートウェイがコンテナ内動作し、システムを保護したままローカル / リモートのエージェントを動かせる。
- **NVIDIA OpenShell on Windows**: NVIDIA の OSS エージェント runtime を MXC 経由で。自律的な常時オン (always-on) エージェントを安全に配布。
- **Agent 365 native integration with MXC (7 月 Preview)**: Defender / Entra / Intune / Purview をローカル agent にも適用。
- **Windows 365 for Agents (GA)**: Cloud PC をエージェントの実行環境に。Entra ID 参加 / Intune 管理 / ポリシー強制。consumption-based pricing。<https://learn.microsoft.com/en-us/windows-365/agents/pricing-paygo-always-available>

### 5.3 オンデバイス AI

- **Aion 1.0 Instruct (Preview)**: 軽量・高速の on-device SLM
- **Aion 1.0 Plan (今後数ヶ月)**: 14B パラメータ・32K コンテキストの reasoning & tool-calling モデル。Windows 同梱、サブエージェント orchestration や file 操作などフル agentic ワークロードをローカル実行。
- **Windows AI API 拡張**: Speech Recognition API を NPU/CPU で、Phi Silica を dGPU に、Video Super Resolution を CPU に。
- **Surface RTX Spark Dev Box**: 1 PFLOPS の AI compute + 128GB unified memory (CPU/GPU 共有)。最大 120B パラメータモデルをローカル実行可能。VS Code, GitHub Copilot プリインストール。<https://www.microsoft.com/devbox/>
- **DGX Station for Windows (Q4 2026)**: NVIDIA GB300 Grace Blackwell Ultra Superchip 搭載のデスクサイド AI スパコン、最大 1T パラメータモデル対応。
- **NVIDIA RTX Spark + N1X**: MAI モデルが N1X に最適化される予定 ([MAI トランスクリプト参照](https://microsoft.ai/news/microsoft-build-2026-mai-keynote-transcript/))。

## 7. AI アプリ開発・GitHub Copilot から見た「意味」のまとめ

### 7.1 アーキテクチャレベルの示唆

- **エージェントが新しいアプリ単位** という前提のもと、Microsoft はランタイム (MAF, Hosted Agents)、コンテキスト (Microsoft IQ / Web IQ)、モデル (MAI ファミリー + Foundry 上の OSS / サードパーティ)、配布 (Microsoft 365 / Teams / Windows Agent Store)、観測・評価・統治 (OpenTelemetry / Adaptive Evaluations / Agent Control Specification) の 5 つを **製品ライン横断のレイヤ**として揃えた。
- 統治と分離は **Windows OS と Entra ID** に降りた (MXC / Agent 365 / Windows 365 for Agents)。エージェントは「**それぞれが独自の Entra アイデンティティを持つ**」モデルとなり、監査と権限管理が "人と同じ仕組み" に乗る。

### 7.2 GitHub Copilot の戦略的位置付け

- GitHub Copilot は **単なるエディタ補完** から、**Microsoft / GitHub 共通の agent runtime + 配布チャネル** へ転換 (Copilot SDK GA / Copilot App / Copilot CLI / VS が SDK ベースに統一)。
- Microsoft Scout や Microsoft Discovery が "GitHub Copilot のライセンスで利用開始" となるなど、**Copilot アカウントが Microsoft AI スタック全体の入口** になりつつある。
- BYOK (OpenAI / Microsoft Foundry / Anthropic) と Microsoft 自社 MAI モデルが並列に乗ることで、**「OpenAI 一本足からの脱却」**が技術 / 商流の両面で進む (MAI-Code-1-Flash の VS Code 既定モデル化、MAI-Thinking-1 の SWE Bench Pro 53% など)。

### 7.3 開発者にとっての実務インパクト

| やりたいこと | Build 2026 で使えるもの |
|---|---|
| アプリにエージェントを組み込む | **GitHub Copilot SDK** (TS/Python/Go/.NET/Rust/Java) |
| 本番運用級のマルチエージェントを構築 | **Microsoft Agent Framework 1.0** + **Hosted Agents (Foundry)** |
| Issue/PR ベースで agentic 開発を回す | **GitHub Copilot App** (Technical Preview) |
| 端末上の agentic ワークフロー | **GitHub Copilot CLI** + **Intelligent Terminal** + **MXC** |
| Windows 上でエージェントを安全に走らせる | **MXC** + **OpenClaw on Windows** + **Windows 365 for Agents** |
| 自社データ / 業務知識でエージェントを強化 | **Microsoft IQ** (Work / Foundry / Fabric / Web) + **Frontier Tuning** |
| ローカルでフロンティアモデル運用 | **Surface RTX Spark Dev Box** / **DGX Station for Windows** + **Aion 1.0 Plan** |
| OpenAI 以外の選択肢 | **MAI ファミリー** (Foundry / Open Router / Fireworks / Baseten) |
