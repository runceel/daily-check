### 主要な変更点

- **13.4.1 patch リリース確定**: [#17819](https://github.com/microsoft/aspire/pull/17819) で patch バージョンを 13.4.1 に bump、[#17817](https://github.com/microsoft/aspire/pull/17817) で `release/13.4` → `main` のマージ、[#17805](https://github.com/microsoft/aspire/pull/17805) で `PackageValidationBaselineVersion` を 13.4.0 に更新。13.4.0 GA → 13.4.1 patch までのフローが 6/2 中に確定
- **DCP 関連リグレッション 2 件を main + 13.4 backport で同時修正**: [#17825](https://github.com/microsoft/aspire/pull/17825) (main) / [#17826](https://github.com/microsoft/aspire/pull/17826) (`release/13.4`) で「explicit-start DCP registration を deferred 化」して [#17813 (WithArgs runs on startup)](https://github.com/microsoft/aspire/issues/17813) を解決。さらに [#17827](https://github.com/microsoft/aspire/pull/17827) で `Fix Redis persistent lifetime startup` を投入 (関連 Issue [#17822](https://github.com/microsoft/aspire/issues/17822))
- **dashboard / telemetry の細かい修正集中**: [#17816](https://github.com/microsoft/aspire/pull/17816) で telemetry filter に timestamp search qualifier 追加、[#17821](https://github.com/microsoft/aspire/pull/17821) で「resource が現れるまで streaming telemetry を待つ」修正 (関連 [#17802](https://github.com/microsoft/aspire/issues/17802) クローズ)
- **VS Code 拡張 E2E (ExTester) 基盤投入**: [#17772](https://github.com/microsoft/aspire/pull/17772) `Add VS Code extension E2E test architecture` がマージ ([#17727](https://github.com/microsoft/aspire/issues/17727) クローズ)。あわせて [#17843](https://github.com/microsoft/aspire/pull/17843) で拡張のブランディングを「Aspire」へ統一、[#17839](https://github.com/microsoft/aspire/pull/17839) で telemetry イベント名修正
- **proxyless endpoint allocation の最終形**: [#17851](https://github.com/microsoft/aspire/pull/17851) `Add proxyless endpoint on-demand allocation` がマージ。proxyless モードで動的にエンドポイント割当が必要なシナリオを解消
- **VS Code 拡張 resource commands**: [#17698](https://github.com/microsoft/aspire/pull/17698) `Display resource commands in VS Code extension tree view` がマージ ([#17667](https://github.com/microsoft/aspire/issues/17667) クローズ)
- **AppHost dotnet run 委譲**: [#17748](https://github.com/microsoft/aspire/pull/17748) `Delegate bundled AppHost dotnet run to Aspire CLI` がマージ。`aspire run` の挙動を AppHost 経由でも一貫させる方向
- **CLI 周辺の品質改善**: [#17830](https://github.com/microsoft/aspire/pull/17830) で長い CLI E2E テスト名を短縮 + 記録コメントテーブル改善、[#17829](https://github.com/microsoft/aspire/pull/17829) で LOC breakdown を表示する canvas extension を追加
- **CLI npm 配布の release integration 完成**: 前回ウォッチ対象だった [#17766](https://github.com/microsoft/aspire/pull/17766) (`[release/13.4] Add Aspire CLI npm package release integration`) がマージ
- **新規重要 Issue の山**: [#17822](https://github.com/microsoft/aspire/issues/17822) Redis persistent lifetime 起動失敗 (#17827 で main 修正済み)、[#17831](https://github.com/microsoft/aspire/issues/17831) Azure Storage Emulator も同じ persistent lifetime で起動失敗、[#17847](https://github.com/microsoft/aspire/issues/17847) Postgres `PublishAsMigrationBundle` 失敗、[#17836](https://github.com/microsoft/aspire/issues/17836) NixOS 上で .NET 検出失敗、[#17837](https://github.com/microsoft/aspire/issues/17837) `aspire update --self` に version 指定、[#17838](https://github.com/microsoft/aspire/issues/17838) `aspire deploy` のリソースグループタグ固定化問題、[#17842](https://github.com/microsoft/aspire/issues/17842) `apphost.ts` → `apphost.mts` への migration warning 提案
