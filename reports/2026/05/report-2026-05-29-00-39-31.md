- 作者: lokitoth / 状態: merged / マージ日時 (UTC): `2026-05-29 00:28:19`

**変更概要**

前回レポートで「Orchestration が GA 相当の API になる節目」としてウォッチしていた PR が、今回ウィンドウでマージされた。`Microsoft.Agents.AI.Workflows` 配下の各 Orchestration ビルダー / 関連型から `[Experimental]` 属性を一括除去するもので、コード変更は実質「属性行の削除のみ」(+1/-28、9 ファイル、1 コミット)。これにより Sequential / Concurrent / Handoff / GroupChat / Magentic の各オーケストレーションパターンが、experimental 警告 (コンパイル時の `EXTEXP****` 診断) なしで利用できる安定 API となった。**破壊的変更ではない** (PR でも breaking ではないと明示)。

**変更ファイル (主要)**

```

dotnet/src/Microsoft.Agents.AI.Workflows/HandoffWorkflowBuilder.cs          (+0/-7)

```

**コミットレベルの詳細**

- `AgentWorkflowBuilder` / `HandoffWorkflowBuilder` / `MagenticWorkflowBuilder` の各ビルダー API から `[Experimental]` が外れ、**Orchestration 構築 API が安定 API に昇格**。これらを参照する利用者側は、これまで必要だった `#pragma warning disable EXTEXP****` や `<NoWarn>` 指定を削除できる。
- Magentic 関連の公開型 (`MagenticPlanReviewRequest` / `MagenticPlanReviewResponse` / `MagenticProgressLedger`) からも experimental 属性が除去され、Magentic プラン審査ワークフローの型が正式 API 化。
- `HandoffAgentExecutor` / `HandoffMessagesFilter` / `MagenticOrchestrator` といった Specialized 実装側の属性も整理。シグネチャ変更・型の追加削除・既定値変更は一切なく、**純粋に安定性表明の変更**。

```csharp
// Before: experimental 警告の抑制が必要だった
#pragma warning disable MEAI001 // (experimental Orchestration API)
var workflow = AgentWorkflowBuilder.BuildSequential(agentA, agentB);
#pragma warning restore MEAI001

// After: 警告抑制なしでそのまま利用可能 (安定 API)
var workflow = AgentWorkflowBuilder.BuildSequential(agentA, agentB);
```

**関連 Issue / Discussion**

- 前回レポートの継続ウォッチ項目 (#6164) のクローズに相当。Python 側の `[Experimental]` 整理や Hosting.Channels (.NET #6151) とは独立。

**既存利用者への影響**

マイグレーション作業は不要。むしろ既存利用者は experimental 警告の抑制コードを撤去できる「クリーンアップ可能」な変更。API シグネチャは不変のため、再ビルドだけで追随できる。
