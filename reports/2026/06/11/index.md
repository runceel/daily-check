## エグゼクティブサマリー

- [microsoft/agent-framework](./agent-framework.md) は Python MCP tool sampling に [破壊的変更](https://github.com/microsoft/agent-framework/pull/6413) が入りました。sampling 承認ガードレール追加により、MCP sampling 利用コードは承認/拒否フローの見直しが必要です。
- [microsoft/aspire](./aspire.md) は obsolete/removal 系の変更が複数あります。`PublishAsConnectionString`、context の `ServiceProvider`、`aspire exec` を使う AppHost 拡張や自動化スクリプトは移行確認が必要です。
- [dotnet/aspnetcore](./aspnetcore.md) は Razor Components の antiforgery/CSRF middleware 委譲、DBSC、NuGet 脆弱性監査警告のエラー化、`Microsoft.AspNetCore.Grpc.Swagger` 非推奨化が継続ウォッチ対象です。
- [Azure 更新](./azure.md) では Azure SQL Database の Microsoft Entra server principals が GA になった一方、Azure Synapse Link for Azure Cosmos DB NoSQL と Azure VPN Client for Linux (Preview) の廃止が告知されました。
- [GitHub Changelog](./github-changelog.md) では Copilot CLI の `/security-review` がパブリック プレビューとなり、CLI からの事前セキュリティ確認フローに組み込めるようになりました。

## 主要トレンド

今回は「認証・セキュリティ強化」と「既存 API/コマンドの整理」が横断テーマです。Azure SQL の Entra login GA、ASP.NET Core の DBSC/NU190x、Copilot CLI の security review は安全性を高める方向の更新です。
一方で Aspire と ASP.NET Core では obsolete/非推奨/破壊的変更候補が目立ち、既存自動化・AppHost 拡張・テンプレート構成の棚卸しが必要です。
Agent Framework と Aspire は coding agent や Copilot 統合の周辺整備も進んでおり、MCP sampling、GitHub Copilot SDK、telemetry、issue triage など agent 運用の実用面が強化されています。
