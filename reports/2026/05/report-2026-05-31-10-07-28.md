##### [#6156](https://github.com/microsoft/agent-framework/pull/6156) — .NET: Forward Magentic participant replies to manager

- 作者: @ghominejad / 状態: merged / マージ日時 (UTC): `2026-05-29 22:00:41`
- ラベル: bug / .NET

**変更概要**

.NET の `MagenticOrchestrator` が参加エージェントの返信を manager のチャット履歴に戻していなかったため、manager が進捗台帳 (progress-ledger) プロンプトを `MagenticTaskContext.ChatHistory` から組み立てる際に参加者の返信を一切見られず、同じ話者を選び続けて `MaxRounds` で打ち切られていた。Python の `_MagenticOrchestratorExecutor._handle_response` が行っている `chat_history.extend(messages)` 相当の処理を .NET 側に復元し、パリティを回復する。Issue #6172 / #6170 を修正。

**変更ファイル (主要)**

```
MagenticOrchestrator.cs (TakeTurnAsync の subsequent-turn 分岐)
MagenticOrchestrationTests.cs (回帰テスト追加)
RecordingReplayAgent.cs (テストヘルパー追加)
```

**コミットレベルの詳細**

- `MagenticOrchestrator.TakeTurnAsync` の **2 ターン目以降の分岐**で、受け取った `messages` (参加エージェントの返信) を `RunCoordinationRoundAsync` 呼び出し前に `_taskContext.ChatHistory` へ追加するよう修正。
- 初回ターンや終了済みコンテキストの挙動は変更なし。**公開 API の変更なし**。
- 回帰テスト `RunCoordinationRound_Forwards_Participant_Reply_To_ManagerAsync` を追加。修正前は確実に失敗することを確認済み。

**関連 Issue / Discussion**

- [#6172](https://github.com/microsoft/agent-framework/issues/6172) / [#6170](https://github.com/microsoft/agent-framework/issues/6170) (重複起票、いずれもクローズ)

**既存利用者への影響**

`MagenticOrchestrator` を使っているマルチエージェントワークフローでは、これまで早期に `MaxRounds` で打ち切られていたケースが正しく収束するようになる。マイグレーション作業は不要。

- 作者: @nicoleserafino / 状態: merged / マージ日時 (UTC): `2026-05-29 22:00:41`
- ラベル: .NET / bug

**変更概要**

ワークフロー遷移時に `CreatedAt` がプロパティの自己代入になっていたため、上流ステップで `message.CreatedAt` が設定されていてもワークフローエンティティに反映されず失われていた。
